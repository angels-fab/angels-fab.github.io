A~D 메인 트랙 완주 (2026-07-12)

계정 체계를 만들고(A), 권한을 나누고(B), 포털이 사람마다 다르게 동작하도록(C·D) 바꿨습니다. 개인화 항목은 전부 내 계정에만 적용 — 팀 공유 데이터에는 영향이 없습니다.

A 계정 기본 B 권한 4단계 C 개인화 1차 D 개인화 2차
A

계정 기본각자 계정으로 안전하게 드나드는 기반

완료 · 7/7
비밀번호 변경
설정에서 본인 비밀번호를 직접 변경. 이전엔 초기 비밀번호를 그대로 쓸 수밖에 없었음.
가입 신청 → 관리자 승인
새 팀원이 사번으로 가입 신청하면 관리자가 설정의 "가입 승인 대기"에서 승인. 승인 전에는 아무것도 못 보는 대기 상태.
가입 오류(이메일 블로커) 해결
사번 계정은 받을 메일함이 없어 확인 메일 단계에서 가입이 막히던 문제 → 메일 확인 절차를 꺼서 해결.
B

권한 4단계게스트 · 유관자 · 팀원 · 관리자

뼈대 완료 · 살은 보류
역할 4단계 골격 7/7
게스트(로드맵·행사·바로가기만) → 유관자(+로그인) → 팀원(팀 콘텐츠 전체 열람 + 장비 열람) → 관리자(+사용자 관리). 역할별로 메뉴와 화면이 자동으로 달라짐.
사용자 관리 화면 7/7
설정에서 회원 목록 · 권한 변경 · 강퇴 가능(본인 계정 잠금 방지 포함).
보류분 — 조건 생기면 착수
팀원 글쓰기 권한(팀원이 실제로 쓸 일이 생기면) · 유관자 장비 제한열람(초대할 때) · 계정 이력장부(추적할 일 생기면).
C

개인화 1차각자 보던 화면 · 배지 · 순서

완료 · 7/12
Stage 1 — 화면 기억 7/7나에게만
캘린더 뷰(월/주/목록)·업무현황 탭을 계정별 서버 저장 — 사무실 PC에서 설정한 게 집·폰에서도 그대로. 이후 모든 개인화가 재사용하는 기초 공사.
Stage 2 — 내 기준 새 글 배지 + 필터 기억 7/12나에게만
메뉴의 새 글 숫자가 "내가 안 본 글"만 카운트 — 페이지에 들어가면 자동 읽음 처리, 이후 올라온 글만 다시 셈. 업무현황 구분·담당자 필터도 기억. 내부적으로 설정 저장을 키 단위 병합으로 바꿔 여러 기기·탭이 서로 설정을 지우던 위험도 함께 제거.
Stage 3 — 업무 카드 순서 개인별 7/12나에게만
카드를 드래그해 바꾼 순서가 내 계정에만 저장 — 팀원 화면 순서는 불변. 한 번도 안 바꾼 사람은 기존 공용 순서 그대로(화면 무변화). 상태 이동·삭제는 계속 팀 공유.
D

개인화 2차홈 배치 · 관심 업무

완료 · 7/12
홈 화면 배치 개인화 나에게만
홈 "운영 대시보드" 제목 옆 조절 아이콘 → 섹션(운영 KPI·일정·관심 업무·업무 현황·장비 현황·공지사항)의 순서 변경·숨김. "기본 배치로" 한 번에 복원. FAB 구축 로드맵은 항상 최상단 고정.
관심 업무 별(★) 나에게만
업무 카드의 별을 켜면 홈 "관심 업무" 섹션에 고정. 홈에서 다시 누르면 해제, 하나도 없으면 섹션 자체가 안 보임. (Remind 압정과 혼동되지 않게 별 아이콘.)
새 기능 안내 팝업 (후속)
로그인하면 위 기능들을 소개하는 안내가 1회 표시. "다시 보지 않기" 체크 + 확인한 사람에게만 사라지고, 그냥 닫으면 다음 접속 때 다시 안내. 새 기능 배포 시 전원에게 재안내 가능한 구조.

품질 검증 (C Stage 2 ~ D)

  • 단계마다 실제 로그인 실측 — 배지 감소/부활, 필터·순서 저장/복원, 팀 데이터 불변까지 DB로 확인
  • 멀티에이전트 적대적 리뷰 3회(총 21개 에이전트) — 확정 결함 13건 발견, 전부 수정 후 배포
  • DB 마이그레이션 1건(설정 키 단위 병합 저장) — 여러 기기 동시 사용 시 설정 유실 위험 구조적 제거
  • 매 커밋 타입체크·빌드 통과 · 자동 배포
